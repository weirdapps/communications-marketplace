## Core Style Rules

### 1. BREVITY
- Define your typical reply length (e.g., 1-10 words, 1-2 sentences, full paragraphs)
- Set the default verbosity level

### 2. TWO FORMATS: BRIEF vs FULL

**BRIEF replies (default):**
- No greeting, no closing
- Straight to the point
- Example: "approved", "let's discuss tomorrow"

**FULL emails (longer, substantive messages):**
- Opens with greeting (e.g., "Hi team," or "Dear [name],")
- Body content
- Closing (e.g., "Thanks," or "Best,")
- Your name

**When to use FULL format:**
- Strategic communications to multiple people
- Longer emails (3+ sentences)
- External communications

### 3. LANGUAGE
- Define your primary language
- Define when you switch languages (e.g., English for international contacts)

## Reply Patterns (by type)

### Type 1: APPROVAL / AGREEMENT
Examples of how you approve things:
- "sounds good"
- "approved"
- "go ahead"

### Type 2: DELEGATION / REDIRECT
How you hand off tasks:
- "@[Name] can you take this?"
- "[Name], please handle"

### Type 3: POSITION / DECISION
How you state decisions:
- Direct statement of position
- Brief reasoning if needed

### Type 4: THANK-YOU
How you thank people:
- "thanks [name]"
- "appreciated"

### Type 5: QUALITY FEEDBACK
How you give constructive criticism:
- Direct, specific feedback
- Action-oriented

### Type 6: FORWARDING
How you forward emails:
- No text (just FW)
- Or: "FYI"

## Decision Flow for Drafting

```
INCOMING EMAIL
  │
  ├─ Is it informational/FYI?
  │   → No reply needed. Skip.
  │
  ├─ Does it ask for approval/decision?
  │   → Give the decision directly
  │
  ├─ Does it ask you to do something?
  │   → Delegate if possible, or acknowledge
  │
  ├─ Is it an escalation/problem?
  │   → Assign ownership or ask for info
  │
  └─ Is it a thank-you/acknowledgment?
      → Short acknowledgment
```
